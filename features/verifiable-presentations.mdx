The AgrospAI Portal integrates Gaia-X verifiable presentations to provide transparency about data compliance, provenance, and trustworthiness. Users can view and verify credentials associated with assets and participants.

## Overview

Verifiable presentations are cryptographically-signed credentials that attest to specific claims about:

- **Data assets** - Compliance, terms, and characteristics
- **Service providers** - Identity and capabilities
- **Participants** - Legal status and compliance

The portal uses the Gaia-X compliance framework (API version 2210) to verify these credentials.

## What Are Verifiable Presentations?

A verifiable presentation is a collection of one or more verifiable credentials presented together. In the context of the AgrospAI Portal:

- **Verifiable Credential (VC)**: A signed statement about a subject (e.g., "This organization is GDPR compliant")
- **Verifiable Presentation (VP)**: A collection of VCs presented by a holder to a verifier

## Verification Process

The portal automatically verifies presentations when displayed.

### Verification Component

From `src/components/VerifiablePresentation/VerifiablePresentationVerification/index.tsx`:

```typescript
export const VerifiablePresentationVerification = ({
  verifiablePresentation,
  index,
  className
}: Readonly<VerifiablePresentationVerificationProperties>) => {
  const { address } = useVerifiablePresentationContext()
  const {
    data: { verified, idMatch, isIdMatchVerifiable, complianceApiVersion } = {},
    isLoading
  } = useVerifiablePresentationVerification(
    verifiablePresentation,
    address,
    index
  )

  return (
    <VerifiedBadge
      name="Presentation"
      idMatch={idMatch}
      isValid={verified}
      isIdMatchVerifiable={isIdMatchVerifiable}
      apiVersion={complianceApiVersion}
      isLoading={isLoading}
      timestamp="exact"
      className={className}
    />
  )
}
```

### Verification Criteria

The verification hook checks:

1. **Cryptographic Signature**: Validates that the credential was signed by a trusted issuer
2. **ID Match**: Verifies the credential subject matches the claimed identity
3. **API Version**: Confirms compliance with the expected Gaia-X API version
4. **Expiration**: Checks credential is still valid (not expired)
5. **Revocation Status**: Ensures credential hasn't been revoked

## Fetching Verifiable Presentations

Presentations are fetched from the compliance API.

### API Integration

From `src/@utils/verifiablePresentations/api.ts`:

```typescript
const API = axios.create({
  baseURL: '/api',
  timeout: 2000
})

export const getVerifiablePresentations = async (
  address: Address,
  signal?: AbortSignal
): Promise<GaiaXVerifiablePresentationArray> =>
  address
    ? API.get(CredentialRoutes.GetPresentation(address), {
        signal
      })
        .then(({ data }) => data)
        .catch((err) => {
          console.error('Error fetching verifiable credentials', err)
          return []
        })
    : []
```

### Compliance Configuration

The compliance API is configured in `app.config.js`:

```javascript
complianceUri:
  process.env.NEXT_PUBLIC_COMPLIANCE_URI ||
  'https://www.delta-dao.com/compliance',

complianceApiVersion:
  process.env.NEXT_PUBLIC_COMPLIANCE_API_VERSION || '2210',
```

## Credential Types

### Service Credentials

Attest to characteristics of a data service:

- **Service Offering**: What the service provides
- **Terms & Conditions**: Contractual terms
- **Data Protection**: GDPR and privacy measures
- **Data Processing**: How data is handled

### Participant Credentials

Attest to characteristics of organizations or individuals:

- **Legal Person**: Legal entity status
- **Legal Registration**: Business registration details
- **Terms & Conditions**: Participant agreement
- **Data Protection Officer**: Contact information

### Data Resource Credentials

Attest to characteristics of data assets:

- **Data Resource**: Asset description
- **Exposed Through**: Service endpoints
- **Produced By**: Data provenance
- **Copyrighted By**: Intellectual property rights

## Credential Validation

### Finding Specific Credential Types

Utility function from `src/@utils/verifiablePresentations/utils.ts`:

```typescript
export const findVCType = (
  credentials: GaiaXVerifiablePresentation[],
  target: GaiaXCredentialSubjectType
): GaiaXVerifiableCredential[] =>
  credentials.map(
    (c) =>
      c.verifiableCredential.filter(
        (vc) => vc.credentialSubject.type === target
      )[0]
  )
```

This allows filtering credentials by type for specific use cases.

This ensures the portal remains functional even if credential services are unavailable.
