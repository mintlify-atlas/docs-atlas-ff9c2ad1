## Network Configuration Fields

### Basic Configuration

| Field | Type | Description |
|-------|------|-------------|
| `chainId` | number | Unique blockchain network identifier |
| `isDefault` | boolean | Whether this network is enabled by default |
| `isCustom` | boolean | Indicates a custom/non-standard EVM chain |
| `network` | string | Human-readable network name |
| `oceanTokenSymbol` | string | Symbol for Ocean Protocol token |

### Smart Contract Addresses

| Field | Description |
|-------|-------------|
| `oceanTokenAddress` | Ocean Protocol ERC20 token contract |
| `nftFactoryAddress` | Factory contract for creating NFTs |
| `fixedRateExchangeAddress` | Contract for fixed-price asset exchanges |
| `dispenserAddress` | Contract for dispensing free datatokens |
| `opfCommunityFeeCollector` | Ocean Protocol Foundation fee collector |

### Transaction Parameters

| Field | Type | Description |
|-------|------|-------------|
| `startBlock` | number | Block number to start indexing from |
| `transactionBlockTimeout` | number | Number of blocks before transaction times out |
| `transactionConfirmationBlocks` | number | Required confirmations for transactions |
| `transactionPollingTimeout` | number | Polling timeout in milliseconds |
| `gasFeeMultiplier` | number | Multiplier for gas fee estimation (e.g., 1.1 = 10% buffer) |

### Service Endpoints

| Endpoint | Purpose |
|----------|----------|
| `providerUri` | Ocean Provider service for asset access |
| `providerAddress` | Wallet address of the Provider service |
| `metadataCacheUri` | Aquarius metadata cache for asset discovery |
| `nodeUri` | RPC endpoint for blockchain interactions |
| `subgraphUri` | The Graph endpoint for indexed data queries |
| `explorerUri` | Block explorer URL for transaction viewing |

## Adding a New Network

To add support for a new blockchain network:

1. **Add chain configuration** to `chains.config.js`:

```javascript
{
  chainId: YOUR_CHAIN_ID,
  isDefault: false,
  isCustom: true,
  network: 'your-network-name',
  // ... other required fields
}
```

2. **Add network metadata** to `networksMetadata.config.js`:

```javascript
{
  chainId: YOUR_CHAIN_ID,
  name: 'Your Network Name',
  chain: 'YourChain',
  rpc: ['https://rpc.your-network.com'],
  nativeCurrency: {
    name: 'Token Name',
    symbol: 'SYMBOL',
    decimals: 18
  },
  // ... other metadata
}
```

3. **Update app.config.js** if needed to include the new chain ID in supported chains.

4. **Test thoroughly** on the new network before deploying to production.

## Usage in Components

### Get Current Network

```tsx
import { useNetwork } from 'wagmi'
import { chains } from '@/../chains.config'

function NetworkDisplay() {
  const { chain } = useNetwork()
  const config = chains.find(c => c.chainId === chain?.id)
  
  return (
    <div>
      <h3>{config?.network}</h3>
      <p>Chain ID: {config?.chainId}</p>
      <a href={config?.explorerUri}>View Explorer</a>
    </div>
  )
}
```

### Access Network Metadata

```tsx
import { useNetworkMetadata } from '@hooks/useNetworkMetadata'

function NetworkInfo() {
  const { networkData, networkName } = useNetworkMetadata()
  
  return (
    <div>
      {networkName}
      {networkData?.nativeCurrency.symbol}
    </div>
  )
}
```

## Best Practices

<Tip>
  Always test contract interactions on Devnet before deploying to Testnet or production.
</Tip>

<Warning>
  Ensure RPC endpoints are reliable and have sufficient rate limits for your expected traffic.
</Warning>

<Info>
  Keep the `startBlock` value updated to improve indexing performance on new deployments.
</Info>

## Troubleshooting

### Network Connection Issues

1. **Check RPC endpoint availability**: Verify the `nodeUri` is accessible
2. **Verify chain ID**: Ensure wallet is connected to correct network
3. **Check block explorer**: Confirm network is operational

### Transaction Failures

1. **Increase `gasFeeMultiplier`**: If transactions fail due to low gas
2. **Check `transactionBlockTimeout`**: Increase if network is congested
3. **Verify contract addresses**: Ensure all addresses are correct for the network
