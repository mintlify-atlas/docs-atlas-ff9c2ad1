The AgrospAI Portal uses multiple JavaScript configuration files to customize the application behavior, marketplace settings, and supported chains.

## Configuration Files

The main configuration files are:

- `app.config.js` - Main application configuration
- `chains.config.js` - Blockchain network configurations
- `address.config.js` - Featured assets and whitelists
- `networksMetadata.config.js` - Network metadata for UI display

## app.config.js

The main configuration file for the application. Located at the root of the project.

### Core Settings

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `metadataCacheUri` | string | `https://aquarius.pontus-x.eu` | URI of the metadata cache (Aquarius) instance for all networks |
| `complianceUri` | string | `https://www.delta-dao.com/compliance` | URI for compliance verification service |
| `complianceApiVersion` | string | `2210` | API version for compliance service |
| `chainIds` | number[] | from `getDefaultChainIds()` | List of default chain IDs for metadata cache queries |
| `chainIdsSupported` | number[] | from `getSupportedChainIds()` | List of all supported chain IDs |
| `customProviderUrl` | string | from env | Custom provider URL for development |
| `infuraProjectId` | string | `xxx` | Infura project ID |

### Marketplace Configuration

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `marketFeeAddress` | string | `0x9984b2453eC7D99a73A5B3a46Da81f197B753C8d` | ETH address where marketplace fees are sent |
| `publisherMarketOrderFee` | string | `0` | Absolute fee taken upon ordering (set on ERC20 creation) |
| `publisherMarketFixedSwapFee` | string | `0` | Percentage fee for fixed rate exchange (publisher) |
| `consumeMarketOrderFee` | string | `0` | Absolute fee taken upon ordering (consumer) |
| `consumeMarketFixedSwapFee` | string | `0` | Percentage fee for fixed rate exchange (consumer) |
| `defaultDatatokenTemplateIndex` | number | `2` | Default datatoken template index |
| `defaultTokenSymbol` | string | `EURAU` | Token symbol used as fallback in UI |

### Pricing Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `allowFixedPricing` | string | `true` | Show fixed pricing tab to publishers |
| `allowDynamicPricing` | string | `false` | Show dynamic pricing tab to publishers |
| `allowFreePricing` | string | `true` | Show free pricing tab to publishers |

### Privacy & Compliance

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `defaultPrivacyPolicySlug` | string | `/privacy/en` | Default privacy policy markdown file slug |
| `privacyPreferenceCenter` | string | `false` | Enable GDPR privacy preference center |
| `defaultTermsAndConditionsUrl` | string | `https://portal.agrospai.udl.cat/terms` | Default terms and conditions URL |

### External Services

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `purgatoryUrl` | string | empty | Purgatory URI for asset blacklisting (empty disables API call) |
| `dockerHubProxyUrl` | string | `https://dockerhub-proxy.delta-dao.com` | URL to fetch Docker Hub image info |
| `plausibleDataDomain` | boolean | `false` | Plausible analytics data domain |

### Automation Configuration

```javascript
automationConfig: {
  enableAutomation: process.env.NEXT_PUBLIC_ENABLE_AUTOMATION || 'true',
  networkTokenFundDefaultValue: '2',
  erc20ApprovalDefaultValue: '50',
  roughTxGasEstimate: 0.01,
  defaultMode: 'simple'
}
```

| Option | Description |
|--------|-------------|
| `enableAutomation` | Enable/disable automation features |
| `networkTokenFundDefaultValue` | Default value for network token funding |
| `erc20ApprovalDefaultValue` | Default value for ERC20 approvals |
| `roughTxGasEstimate` | Rough estimate for transaction gas |
| `defaultMode` | Default automation mode |

### Chain Configuration Fields

| Field | Type | Description |
|-------|------|-------------|
| `chainId` | number | Blockchain network chain ID |
| `isDefault` | boolean | Whether this chain is enabled by default |
| `isCustom` | boolean | Whether this is a custom (non-standard) chain |
| `network` | string | Network identifier |
| `oceanTokenSymbol` | string | Symbol for Ocean token on this chain |
| `oceanTokenAddress` | string | Ocean token contract address |
| `nftFactoryAddress` | string | NFT factory contract address |
| `fixedRateExchangeAddress` | string | Fixed rate exchange contract address |
| `dispenserAddress` | string | Dispenser contract address |
| `opfCommunityFeeCollector` | string | Community fee collector address |
| `startBlock` | number | Block to start indexing from |
| `transactionBlockTimeout` | number | Blocks to wait before transaction timeout |
| `transactionConfirmationBlocks` | number | Confirmations required |
| `transactionPollingTimeout` | number | Polling timeout in milliseconds |
| `gasFeeMultiplier` | number | Gas fee multiplier for transactions |
| `providerUri` | string | Provider service endpoint |
| `providerAddress` | string | Provider wallet address |
| `metadataCacheUri` | string | Aquarius metadata cache endpoint |
| `nodeUri` | string | RPC node endpoint |
| `subgraphUri` | string | The Graph subgraph endpoint |
| `explorerUri` | string | Block explorer URL |

## address.config.js

Configures featured assets and whitelists.

### Structure

```javascript
module.exports = {
  whitelists: {
    'nft.owner': [],           // Whitelisted NFT owners
    'datatokens.address': []   // Whitelisted datatoken addresses
  },
  featured: [
    {
      title: 'Feature Title',
      assets: [
        'did:op:...',           // Array of DIDs to feature
      ]
    }
  ],
  verifiedAddresses: PONTUSX_ADDRESSES  // Imported from pontusxAddresses.json
```

## Usage in Components

### Accessing App Config

```tsx
import { useMarketMetadata } from '@context/MarketMetadata'

function MyComponent() {
  const { appConfig } = useMarketMetadata()
  
  return <div>{appConfig.metadataCacheUri}</div>
}
```

### Accessing Chain Config

```tsx
import { chains } from '@/../chains.config'
import { useNetwork } from 'wagmi'

function NetworkInfo() {
  const { chain } = useNetwork()
  const chainConfig = chains.find(c => c.chainId === chain?.id)
  
  return <div>{chainConfig?.network}</div>
}
```

## Best Practices

<Tip>
  Always use environment variables for sensitive values and endpoint URLs that may change between environments.
</Tip>

<Warning>
  Don't hardcode contract addresses directly in components. Always reference them from configuration files.
</Warning>

<Info>
  The `app.config.js` prioritizes environment variables, making it easy to override settings per deployment.
</Info>
