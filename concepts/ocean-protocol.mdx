## Overview

Ocean Protocol is a decentralized data exchange protocol that powers the core marketplace functionality of AgrospAI Portal. It enables secure data sharing, tokenization, and monetization while preserving data sovereignty.

<Info>
AgrospAI Portal is built on Ocean Protocol v3.1.3, providing production-ready data exchange capabilities.

## Core Concepts

### Datatokens

Datatokens are ERC-20 tokens that represent access rights to a data asset. Each published dataset has its own unique datatoken.

**Key Properties:**
- One datatoken = one access right to the underlying data
- Can be bought, sold, or transferred
- Exchangeable through fixed-rate exchanges or dispensers

Source: `README.md:107`

### DDO (Decentralized Data Object)

A DDO is a JSON document that contains all metadata and service definitions for a data asset.

```typescript
import { Asset, DDO, Service } from '@oceanprotocol/lib'

// Get a specific service from DDO
function getServiceByName(
  ddo: Asset | DDO,
  name: 'access' | 'compute'
): Service {
  return ddo.services.filter((service) => service.type === name)[0]
}
```

### DID (Decentralized Identifier)

Each asset is identified by a unique DID following the format:

```
did:op:[64 hexadecimal characters]
```

**Validation:**
```typescript
function isValidDid(did: string): boolean {
  const regex = /^did:op:[A-Za-z0-9]{64}$/
  return regex.test(did)
}
```

## Access Control

### Credentials System

Assets can have access restrictions:

```typescript
function isAddressWhitelisted(
  ddo: AssetExtended,
  accountId: string
): boolean {
  // All addresses can access if no credentials specified
  if (!ddo.credentials) return true

  const isAllowed = !ddo.credentials.allow?.length ||
    ddo.credentials.allow?.some((credential) => {
      if (credential.type === 'address') {
        return credential.values.some(
          (address) => address.toLowerCase() === accountId.toLowerCase()
        )
      }
      return true
    })

  const isDenied = ddo.credentials.deny?.some((credential) => {
    if (credential.type === 'address') {
      return credential.values.some(
        (address) => address.toLowerCase() === accountId.toLowerCase()
      )
    }
    return false
  })

  return isAllowed && !isDenied
}
```
