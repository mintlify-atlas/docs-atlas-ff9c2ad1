## What are Verifiable Credentials?

Verifiable Credentials (VCs) are cryptographically secure, tamper-proof digital credentials that prove claims about an entity. In the AgrospAI Portal, they are used to establish trust and verify participant identity in accordance with Gaia-X standards.

## Gaia-X Participant Credentials

The portal uses Gaia-X Participant Credentials to verify the identity of organizations participating in the data space.

### Credential Types

The system supports three types of Gaia-X credentials:

1. **Legal Participant** (`gx:LegalParticipant`)
2. **Legal Registration Number** (`gx:legalRegistrationNumber`)
3. **Gaia-X Terms and Conditions** (`gx:GaiaXTermsAndConditions`)

## Verifiable Presentations

A Verifiable Presentation (VP) is a collection of one or more Verifiable Credentials, packaged together and cryptographically signed by the holder.

### Data Structure


### Verification Process

The verification component checks:

1. **Cryptographic validity** - Verifies the credential signature
2. **Identity match** - Ensures the credential matches the claimed identity
3. **Compliance** - Validates against compliance API

```tsx
const {
  data: { verified, idMatch, isIdMatchVerifiable, complianceApiVersion },
  isLoading
} = useVerifiablePresentationVerification(
  verifiablePresentation,
  address,
  index
)
```

## Legal Participant Credentials

### Required Fields

A Legal Participant credential includes:

```typescript
interface LegalParticipant {
  id: string                         // DID of the participant
  type: 'gx:LegalParticipant'
  'gx:legalName': string            // Official legal name
  'gx:name'?: string                // Display name (optional)
  'gx:website'?: string             // Organization website
  'gx:legalRegistrationNumber': {
    id: string                       // URL to registration number credential
  }
  'gx:headquarterAddress': object   // Headquarters location
  'gx:legalAddress': {
    'gx:countrySubdivisionCode': string  // ISO 3166-2 code
    'gx:addressCountryCode'?: string     // ISO 3166-1 alpha-2
    'gx:streetAddress': string
    'gx:postalCode': string
    'gx:locality': string                // City/town
  }
}
```

## Visual Verification

The portal displays verification status with a visual badge:

- **Verified checkmark** - Credential is cryptographically valid
- **Identity match indicator** - Credential matches the claimed address
- **Compliance version** - Shows which Gaia-X spec version was used

## Using Verifiable Presentations

### For Publishers

1. Obtain a Gaia-X Participant Credential from an authorized issuer
2. Upload your credential when creating a service offering
3. The portal validates and displays your verified status

### For Consumers

1. View the verification status of data providers
2. Check credential details in the asset page
3. Make informed decisions based on verified identities
