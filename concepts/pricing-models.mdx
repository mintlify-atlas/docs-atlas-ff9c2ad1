## Overview

AgrospAI Portal supports multiple pricing models for data assets, giving publishers flexibility in how they monetize their data while maintaining sovereignty and control.

## Available Pricing Models

### Fixed Pricing

Fixed pricing allows publishers to set a specific price for access to their data asset.

<Info>
**Enabled by default** - Publishers can set fixed prices in EURAU or other approved tokens.
</Info>

```javascript
allowFixedPricing: process.env.NEXT_PUBLIC_ALLOW_FIXED_PRICING || 'true'
```

Source: `app.config.js:62`

**How it works:**
- Publisher sets a fixed price per datatoken
- Price is enforced through a Fixed Rate Exchange smart contract
- Consumers purchase datatokens at the set price
- 1 datatoken = 1 access right to the data

Relevant documentation: `README.md:388-390`

### Free Pricing

Free pricing enables publishers to share data at no cost (excluding gas fees).

<Info>
**Enabled by default** - Publishers can make their data freely accessible through a dispenser.
</Info>

```javascript
allowFreePricing: process.env.NEXT_PUBLIC_ALLOW_FREE_PRICING || 'true'
```

Source: `app.config.js:64`

**How it works:**
- Datatokens are allocated to a dispenser smart contract
- The dispenser distributes tokens for free to consumers
- Publishers can set a maximum balance limit per address
- Consumers only pay blockchain gas fees

Relevant documentation: `README.md:392-396`

### Dynamic Pricing

Dynamic pricing with automated market makers (AMM) is currently disabled:

```javascript
allowDynamicPricing: process.env.NEXT_PUBLIC_ALLOW_DYNAMIC_PRICING || 'false'
```

Source: `app.config.js:63`

<Warning>
Dynamic pricing is not available in the current portal configuration.
</Warning>

## Pricing Structure

### Access Details

Each asset has access details that include pricing information:

```typescript
interface AccessDetails {
  type: 'fixed' | 'free' | 'NOT_SUPPORTED'
  price: string                    // Price in base token
  templateId: number               // Datatoken template
  addressOrId: string              // Exchange or dispenser ID
  baseToken: TokenInfo            // Payment token info
  datatoken: TokenInfo            // Datatoken info
  isPurchasable?: boolean         // Can be purchased now
  isOwned: boolean                // User already owns access
  validOrderTx: string            // Previous valid order
  publisherMarketOrderFee: string // Publisher's order fee
}
```

Source: `src/@types/Price.d.ts:39-51`

### Determining Pricing Type

The system determines pricing from on-chain data:

```typescript
// Free pricing via dispenser
if (tokenPrice?.dispensers?.length > 0) {
  const dispenser = tokenPrice.dispensers[0]
  accessDetails.type = 'free'
  accessDetails.addressOrId = dispenser.token.id
  accessDetails.price = '0'
  accessDetails.isPurchasable = dispenser.active
}

// Fixed pricing via exchange
if (tokenPrice?.fixedRateExchanges?.length > 0) {
  const fixed = tokenPrice.fixedRateExchanges[0]
  accessDetails.type = 'fixed'
  accessDetails.addressOrId = fixed.exchangeId
  accessDetails.price = fixed.price
}
```

Source: `src/@utils/accessDetailsAndPricing.ts:124-142`

## Purchasing Process

### Fixed Price Purchase

For fixed pricing, consumers:

1. Approve spending of base tokens
2. Call the Fixed Rate Exchange to swap tokens
3. Receive datatokens
4. Exchange datatoken for access

```typescript
// Order with fixed pricing
const orderTx = await order.buy({
  exchangeContract: config.fixedRateExchangeAddress,
  exchangeId: asset.accessDetails.addressOrId,
  orderTxId,
  providerFees,
  consumeMarketSwapFeeAmount,
  consumeMarketSwapFeeAddress
})
```

Source: `src/@utils/order.ts:94-95`

### Free Access

For free pricing, consumers:

1. Call the dispenser contract
2. Receive datatokens for free (pay only gas)
3. Exchange datatoken for access

```typescript
// Order with free pricing
const dispenser = new Dispenser(config.dispenserAddress, signer)
const dispenserTx = await dispenser.dispense(
  asset.accessDetails.datatoken.address,
  '1',
  address
)
```

Source: `src/@utils/order.ts:165-170`

### For Publishers
